# Список задач для сервиса аналитики

## 1. Анализ транзакций
- [ ] **Рассчитывать общую сумму транзакций за определенный период (день, неделя, месяц).**
  - *Решение:* Создать функцию, которая принимает начальную и конечную дату, и агрегирует сумму транзакций за этот период.
- [ ] **Определять среднюю сумму транзакции.**
  - *Решение:* Создать функцию для расчета средней суммы транзакции за определенный период или за все время.
- [ ] **Выявлять самые частые категории транзакций.**
  - *Решение:* Создать функцию, которая группирует транзакции по категориям, подсчитывает их количество и возвращает N самых популярных категорий.
- [ ] **Находить аномальные транзакции (например, слишком крупные или нетипичные для клиента).**
  - *Решение:* Реализовать алгоритм обнаружения аномалий (например, на основе стандартного отклонения или более сложной модели) для выявления транзакций, которые значительно отличаются от обычного поведения клиента.

## 2. Анализ клиентов
- [ ] **Сегментировать клиентов по различным критериям (например, по среднему балансу, количеству транзакций, дате регистрации).**
  - *Решение:* Разработать функцию для сегментации клиентов на основе заданных критериев, создавая определенные когорты.
- [ ] **Рассчитывать "время жизни" клиента (customer lifetime value).**
  - *Решение:* Внедрить модель для оценки общего дохода, который клиент принесет банку за все время сотрудничества.
- [ ] **Выявлять отток клиентов (churn rate).**
  - *Решение:* Создать функцию для расчета коэффициента оттока путем выявления клиентов, у которых не было активности в течение определенного периода.

## 3. Анализ кредитов
- [ ] **Рассчитывать общий портфель активных кредитов.**
  - *Решение:* Создать функцию для суммирования остатков по всем активным кредитам.
- [ ] **Анализировать распределение кредитов по статусам (активные, погашенные, просроченные).**
  - *Решение:* Создать функцию для группировки кредитов по их статусам и подсчета их количества.
- [ ] **Рассчитывать среднюю процентную ставку по кредитам.**
  - *Решение:* Создать функцию для расчета средневзвешенной процентной ставки по всем или выбранным кредитам.
- [ ] **Прогнозировать риски невозврата кредитов (кредитный скоринг).**
  - *Решение:* Разработать или интегрировать модель машинного обучения для прогнозирования вероятности дефолта клиента по кредиту.
- [ ] **Сегментировать кредитный портфель (например, по типу клиентов, по срокам).**
  - *Решение:* Создать функцию для сегментации кредитов по типу клиента, сроку или другим критериям.

## 4. Анализ вкладов
- [ ] **Рассчитывать общую сумму средств на вкладах.**
  - *Решение:* Создать функцию для суммирования остатков на всех депозитных счетах.
- [ ] **Анализировать динамику притока/оттока вкладов.**
  - *Решение:* Отслеживать приток и отток средств с депозитных счетов с течением времени.
- [ ] **Рассчитывать средний срок вклада.**
  - *Решение:* Создать функцию для расчета среднего срока действия депозитных продуктов.
- [ ] **Прогнозировать вероятность закрытия вклада после окончания срока.**
  - *Решение:* Разработать модель для прогнозирования того, закроет ли клиент свой депозитный счет после окончания его срока.
- [ ] **Анализировать структуру вкладов (по срокам, по суммам).**
  - *Решение:* Создать функцию для анализа вкладов по срокам, суммам или другим характеристикам.

## 5. Анализ по геолокации и выявление мошенничества
- [ ] **Определять "домашний" регион клиента.**
  - *Решение:* Создать функцию для определения наиболее частого местоположения транзакций клиента.
- [ ] **Выявлять транзакции из нетипичных регионов.**
  - *Решение:* Создать функцию для выявления транзакций, совершенных далеко от "домашнего" региона клиента.
- [ ] **Анализировать логины из подозрительных IP-адресов.**
  - *Решение:* Создать функцию для проверки IP-адресов входа по черным спискам или выявления необычных паттернов входа.
- [ ] **Создать систему оповещений о подозрительной активности.**
  - *Решение:* Внедрить систему уведомлений (например, по электронной почте, SMS) для оповещения сотрудников или клиентов о подозрительной активности.

## 6. Анализ использования карт
- [ ] **Анализировать популярность использования карт.**
  - *Решение:* Отслеживать количество транзакций по каждой карте или типу карт.
- [ ] **Отслеживать сроки действия карт для проактивного перевыпуска.**
  - *Решение:* Создать функцию для выявления карт, срок действия которых скоро истекает.
- [ ] **Анализировать причины блокировки карт.**
  - *Решение:* Создать функцию для категоризации и подсчета причин блокировки карт.

## 7. Операционная аналитика (банкоматы и отделения)
- [ ] **Анализировать нагрузку на банкоматы и отделения.**
  - *Решение:* Отслеживать объемы транзакций и поток клиентов в различных точках.
- [ ] **Оптимизировать расположение банкоматов.**
  - *Решение:* Использовать данные для определения оптимальных мест для установки новых банкоматов.
- [ ] **Оценивать эффективность работы отделений.**
  - *Решение:* Измерять производительность отделений на основе различных показателей.

## 8. Анализ активности пользователей
- [ ] **Анализировать частоту и время входа пользователей в систему.**
  - *Решение:* Отслеживать, когда и как часто пользователи входят в систему.
- [ ] **Выявлять неактивных пользователей.**
  - *Решение:* Создать функцию для поиска пользователей, которые не входили в систему в течение определенного периода.
- [ ] **Анализировать используемые устройства для входа.**
  - *Решение:* Отслеживать устройства и браузеры, используемые для доступа к системе.

## 9. Отчеты
- [ ] **Создавать ежедневные, еженедельные и ежемесячные отчеты по ключевым метрикам.**
  - *Решение:* Автоматизировать создание отчетов с ключевыми показателями.
- [ ] **Предоставлять возможность выгрузки отчетов в различных форматах (например, CSV, PDF).**
  - *РеAшение:* Реализовать функции для экспорта отчетов в форматы, такие как CSV и PDF.

## 10. Прогнозирование
- [ ] **Прогнозировать объемы транзакций на будущие периоды.**
  - *Решение:* Использовать анализ временных рядов для прогнозирования будущих объемов транзакций.
- [ ] **Прогнозировать возможный отток клиентов.**
  - *Решение:* Использовать предиктивное моделирование для выявления клиентов, подверженных риску оттока.

## 11. Интеграция
- [ ] **Обеспечить интеграцию с другими сервисами (например, для получения данных о курсах валют).**
  - *Решение:* Разработать API-клиенты для получения данных из внешних сервисов, таких как поставщики курсов валют.
